The Knight's Travails

Puzzle

The Knight's Travails

Given a standard 8x8 chessboard, design an application that accepts two squares identified by algebraic chess notation. The first square is the starting position, and the second square is the ending position. Find the shortest sequence of valid moves to take a Knight piece from the starting position to the ending position. Each move must be a legal move by a Knight. For any two squares there may be more than one valid solution.

Input

Must be two squares identified in algebraic chess notation representing the starting and ending positions of the Knight. The two squares are separated by a space.

Output

Must be a list of squares through which the Knight passes, in algebraic chess notation. This must include the ending position, but exclude the starting position.

Example

Example Input: A8 B7

Example Output: C7 B5 D6 B7

Note: For any two squares there may be more than one valid solution, your program need not print this exact solution if it can find another that is equally short.

Summary

  • A standard 8x8 chessboard where each position is indicated in algebraic notation as showed above left
  • A knight’s legal movement is move to a square that is two squares horizontally and one square vertically, or two squares vertically and one square horizontally as showed above right
  • Input: two squares identified in algebraic chess representing the starting and ending positions of the Knight. The two squares are separated by a space. e.g. A8 B7
  • Output: a list of squares through which the Knight passes excluding starting position including ending position. e.g. C7 B5 D6 B7 or B6 C4 A5 B7, named path
  • The output is the shortest path from the starting position to the ending position. There may be more than one path.
  • This object-oriented application (C#) will print one valid correct path as one solution for any given valid input on the windows console but not all the valid paths.
  • There are no other pieces on the board as obstacles

Solutions

Solution One - C# Windows Console(Desktop)

Overview

The C# Console application is made of three parts

The first part is to obtain user’s input of starting position and ending position, and define some parameters such as the chessboard layout, the Knight’s legal movements, and some lists to hold data.

The second part is to use the Flood Fill method to fill the chessboard with values of minimum of steps from starting position to the rest, and save the information of parent nodes and child nodes. From the table below we can tell two important messages that are, one is the minimum steps needed for a Knight moves from A8 to B7 is 4, another is a Knight takes 1 step from A8 to either C7 or B6 meaning the parent node is A8 and the child nodes are C7 and B6. In this case, A8 is the root, B6 is the child of A8, and B6 is the parent of its sub nodes. C7 can have more than one or no child nodes.

The third part is to generate the path from the starting position to the ending position for a given starting position, minimum steps needed and the information of parent and child nodes as gained from the previous step. Besides, I wrote some unit test cases for verifying the results.

What is the Floodfill Algorithm?

  1. initialize the chessboard with the value of -1 of all positions, set step = 0, set the staring position be 0, e.g. A8 = 0
  2. iterate the 8x8 chessboard from left to right top to bottom, eg. A8,B8....H8, A7,B7....H7,.....,..A1,B1...H1.
  3. given the ending position B7, during iteration process if the position of [row][column] is a legal movement of the Knight, and the value of that position[row][column] is -1, we increase the step value to 1, and mark this position to step(which is 1 now). after this loop, we will find, e.g. B6 = 1, and C7 =1
  4. then continue the loop of new step value (1 now) to find child nodes of B6 and C7 and mark the values of those nodes to be a new step value(which is 2 now)
  5. during the iteration process we can draw the diagram of minimum steps needed from the starting position to the ending position, as you can see on "s.png' image in the attachment.
  6. druing this process we also record the parent node and child nodes information into a List(C#) or a Array(PHP), e.g. { {A8,B6,C7}, {B6,xxx}, {C7,xxx}.....{....} } A8 is the parent node, and its child nodes are B6 and C7.B6 is the parent node and its child nodes are xxx. same relationship to the rest.
  7. from step 3, we know from A8 to B7 takes at least 4 steps. From step 6, given ending position B7, I can find the parent node A5. Given ending position A5, I can find the parent node C4. Given ending position C4, I can find the parent node B6. Yes, all the way up to the starting position recording all the nodes visited. Therefore, I have obtained B6 C4 A5 B7.

This solution is implemented on the client side only, meaning no internet is required.

Solution Two - ASP.NET MVP Webpage(Client/Server)

Overview

The second solution is a ASP.NET MVP application, that wil draw a chessboard on the webpage(HTML JS), and C#. Basically, JavaScript in HTML will send the parameter 'start=A8&end='B7' to the ASP.NET server via AJAX call, and C# at the back handling the POST and send the result back to JavaScript via AJAX, the KnockOut is used to bind and update the steps number on the chessboard. When the user firstly clicks on any square, he marks the starting position showed as a green square. When the user secondly clicks on any square, he marks the ending position showed as a red square. Everytime, a new pair of positions will erase the old step numbers showed on the chessboard. This is a more intuitive solution by provding interactive user interface.
